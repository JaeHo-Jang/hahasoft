# 💾 경로 지정 및 설정 가이드 (Path Management Guide)

경로를 다룰 때는 **절대 경로**와 **상대 경로**의 개념을 명확히 이해하는 것이 가장 중요합니다.

## 1. 기본 개념: 절대 vs. 상대 경로

| 구분 | 정의 | 예시 (Mac/Linux) | 예시 (Windows) | 특징 및 사용처 |
| :--- | :--- | :--- | :--- | :--- |
| **절대 경로** (Absolute Path) | 시스템의 루트 디렉토리(`C:\` 또는 `/`)부터 시작하여 파일까지의 완전한 주소. | `/Users/username/Documents/file.txt` | `C:\Users\username\Desktop\file.txt` | 어떤 위치에서 실행하든 항상 같은 파일을 가리키므로, **가장 확실함**. |
| **상대 경로** (Relative Path) | 현재 작업 중인 디렉토리(Current Working Directory)를 기준으로 한 주소. | `./images/avatar.png` 또는 `../data/file.txt` | `.\images\avatar.png` 또는 `..\data\file.txt` | 코드를 옮기거나 실행하는 위치에 따라 경로가 달라지므로, **유연성이 높음**. |

**💡 핵심 기호:**
*   `./`: 현재 디렉토리 (Current Directory)를 의미합니다.
*   `../`: 상위 디렉토리 (Parent Directory)로 한 단계 올라감을 의미합니다.

## 2. 환경별 경로 지정 방법 (실행 가능 정보)

### A. 운영체제(OS) 레벨의 파일 시스템 경로 지정

파일 탐색기나 터미널에서 파일을 찾거나 생성할 때 사용되는 방식입니다.

| OS | 구분자 | 루트 시작 예시 | 상위 디렉토리 이동 명령어 |
| :--- | :--- | :--- | :--- |
| **macOS/Linux** | `/` (슬래시) | `/` 또는 `~` (홈 디렉토리) | `cd ..` (상위 폴더로 이동) |
| **Windows** | `\` (백슬래시) | `C:\` | `cd ..` (명령 프롬프트에서 사용 가능) |

**✅ Tip: 경로 확인 및 변경 명령어**
*   **현재 위치 확인:**
    *   Mac/Linux: `pwd` (Print Working Directory)
    *   Windows: `cd` 또는 `chdir`
*   **디렉토리 이동:**
    *   Mac/Linux/Win: `cd [원하는 경로]`

### B. 프로그래밍 언어(JavaScript/Expo) 레벨의 경로 지정

모바일 게임 개발(React Native, Expo)에서는 파일 시스템 접근 시 OS에 맞는 방식으로 경로를 처리해야 합니다. **직접 문자열로 하드코딩하기보다 라이브러리를 사용하는 것이 필수적입니다.**

**🛠️ 해결책: Path 모듈 사용 (Node.js/JavaScript)**
`path` 모듈을 사용하여 운영체제별 구분자(`\` 또는 `/`)에 관계없이 안전하게 경로를 조합합니다.

```javascript
// ❌ 잘못된 방식 (OS에 따라 오류 발생 가능)
const wrongPath = "images\\" + "avatar" + ".png"; // Windows 환경에서만 작동

// ✅ 올바른 방식: path.join() 사용 (가장 권장됨)
import * as path from 'path';

// 현재 파일 위치와 관계없이 안전하게 경로를 조합합니다.
const correctPath = path.join(process.cwd(), 'assets', 'images', 'avatar.png');

console.log(`안전한 경로: ${correctPath}`);
```

### C. 시스템 변수 레벨의 경로 설정 (환경 설정)

프로그램이 실행될 때 필요한 기본 폴더나 라이브러리 위치를 지정하는 것이 목적이라면, **환경 변수(Environment Variables)**를 사용해야 합니다. 이는 코드 내에서 `const`로 정의하는 것보다 상위 레벨에서 설정을 변경합니다.

**🎯 목표:** 시스템 전체 또는 특정 터미널 세션에 경로를 임시/영구적으로 추가할 때 사용.

| OS | 명령어 (일반적 예시) | 설명 | 지속성 |
| :--- | :--- | :--- | :--- |
| **Mac/Linux** | `export PATH="/usr/local/bin:$PATH"` | `$PATH` 변수에 새로운 디렉토리(`:`)를 추가하여 시스템이 해당 경로에서 프로그램을 찾게 만듭니다. | 세션 단위 (로그아웃하면 사라짐) |
| **Windows CMD** | `set PATH=%PATH%;C:\new\path` | 기존 `%PATH%`에 새 경로를 연결합니다. | 세션 단위 |

**🚀 영구 설정 방법:**
1.  **Mac/Linux:** `.bashrc`, `.zshrc` 파일의 맨 끝에 `export` 명령을 추가하고 `source ~/.zshrc`로 적용합니다.
2.  **Windows:** 시스템 속성 → 환경 변수 편집에서 '사용자 변수' 또는 '시스템 변수'에 직접 경로를 등록합니다.
