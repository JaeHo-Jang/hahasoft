### 2. 타이포그래피 시스템 (Typography System)
*   **핵심 폰트:** **Google Noto Sans KR (또는 유사한 산세리프 계열)** 사용을 원칙으로 합니다.
    *   *선택 이유:* 가독성이 매우 높고, 다양한 기기에서 깨짐 없이 안정적으로 작동하며, 모던하고 세련된 느낌을 주기 때문입니다. 판타지 장르지만 너무 고풍스러우면 오히려 답답해 보일 수 있습니다.
*   **사용 원칙:**
    *   **제목 (Title):** 굵고 시원하게 배치하여 존재감을 과시합니다. (`Noto Sans KR Bold`, 크기 최대)
    *   **핵심 정보 (Key Data):** 가장 눈에 잘 띄는 색상(Accent/Secondary)과 함께 **Semi-Bold**로 강조하여, 유저가 단번에 '숫자'와 '보상량'을 인식하게 만듭니다.
    *   **설명문 (Body Text):** 가독성을 최우선으로 합니다. 간결하고 짧은 문장을 사용하며, 폰트 크기 대비 여백을 충분히 확보합니다.

### 3. 아트 및 비주얼 스타일 (Art Style Direction)
*   **디자인 철학:** **'고급스러움과 동적 피드백(Dynamic Polish)'**
    1.  **UI 요소 처리:** 모든 버튼, 패널, 게이지는 단순 사각형이 아닌, 미묘하게 입체적인 **베벨(Bevel)** 처리를 적용하고 빛을 반사하는 듯한 효과를 주어 '클릭할 만한 가치가 있는' 것처럼 보이게 합니다.
    2.  **애니메이션:** 로딩 화면과 핵심 전환 구간에는 반드시 애니메이션을 적용해야 합니다. (예: 스킬 사용 시 파티클 이펙트, 레벨업 시 크고 화려한 빛의 폭발). 이는 **"강력한 피드백 설계"** 원칙에 따라 유저 만족감을 극대화합니다.
    3.  **아이콘:** 픽셀아트나 지나치게 사실적인 일러스트는 지양하고, **선(Line Art)을 기반으로 명확하게 형태를 잡은, 메탈릭하거나 크리스털 같은 질감**의 아이콘이 적합합니다.

## II. UI/UX 시스템 적용 원칙 (Monetization Hooks Integration)

가장 중요한 부분입니다. 앞서 분석한 페인 포인트를 해소하고 수익을 발생시키는 모든 지점에는 이 원칙이 개입해야 합니다.

### 1. 성장의 정체 구간(The Wall) 과금 유도
*   **UI/UX 구현:** 캐릭터의 성장 게이지가 특정 임계치(예: 80%)에 도달했을 때, **'현재 재료로는 완성할 수 없습니다.'라는 명확한 메시지와 함께 Accent Color를 사용한 '구매 버튼'이 강제적으로 노출**되어야 합니다.
*   **시각적 장치:** 필요한 핵심 재화 아이콘 주변에 빛나는 효과(Glow Effect)를 주어, 유저가 "이것만 있으면 끝난다"고 느끼게 만듭니다.

### 2. 노동 절감/시간 가속 과금 유도 (The Time Saver)
*   **UI/UX 구현:** 반복적인 파밍 화면에서 '자동화' 또는 '건너뛰기(Skip)' 버튼을 가장 눈에 잘 띄는 위치(하단 중앙, Accent Color 사용)에 배치합니다. 이 버튼은 **시간 단축이라는 보상과 연동된 명확한 메시지**를 전달해야 합니다.
*   **감정적 연결:** "이 모든 걸 수작업으로 하실 건가요? ✨ 프리미엄 패스로 시간을 아껴보세요." 와 같은 문구 배치로 심리적 결핍을 자극합니다.

### 3. 경쟁 과열 시스템(The Status Symbol) 강화
*   **UI/UX 구현:** 랭킹 전용 보상이나 스킨 아이템은 **전용의 특별한 UI 프레임워크**를 사용해야 합니다. 일반적인 장비창과 분리되어, 금속성 질감이나 황금색 테두리로 감싸서 '특별함'을 극대화합니다.
*   **피드백:** 랭킹 상위권을 달성할 때마다 시각적 파티클 이펙트가 폭발하며, **모든 유저에게 '이 사람이 특별하다'는 것을 과시하는 듯한 연출**을 필수적으로 넣습니다.

## II. 💾 핵심 데이터 모델링 스키마 (Schema Definition)
가장 민감한 재화와 성장 데이터를 담을 DB 테이블 구조를 정의합니다. 여기서는 PostgreSQL을 가정하고 설계했습니다.

### 1. `user_profile` (사용자 기본 정보)
*   **Purpose:** 계정 식별 및 사용자 메타데이터 저장.
*   **Fields:**
    ```sql
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    level INT DEFAULT 1,
    last_login TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_premium BOOLEAN DEFAULT FALSE -- 유료 전환 여부
    ```

### 2. `currency_balance` (재화 및 자원 보유량)
*   **Purpose:** 사용자가 보유한 모든 재화를 실시간으로 관리. **가장 트랜잭션이 많은 테이블.**
*   **Fields:**
    ```sql
    user_id INT REFERENCES user_profile(id),
    currency_type VARCHAR(50) NOT NULL, -- 예: 'DIAMOND', 'GOLD', 'ENERGY'
    balance BIGINT DEFAULT 0,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, currency_type)
    ```

### 3. `character_stats` (캐릭터 성장 상태)
*   **Purpose:** 캐릭터의 능력치와 레벨을 저장하며, '성장 정체'를 감지하는 핵심 기준이 됩니다.
*   **Fields:**
    ```sql
    user_id INT REFERENCES user_profile(id),
    current_level INT DEFAULT 1,
    combat_power INT DEFAULT 0, -- 전투력 (가장 중요한 지표)
    available_exp BIGINT DEFAULT 0,
    max_boost_tier INT DEFAULT 1, -- 현재 도달 가능한 강화 최고 단계
    PRIMARY KEY (user_id)
    ```

### 4. `transaction_log` (거래 기록 및 회계 장부)
*   **Purpose:** 모든 재화의 증감 내역을 추적하여 부정 행위(Anti-Cheat) 방지 및 결제 정산에 사용. **절대 삭제되어서는 안 됩니다.**
*   **Fields:**
    ```sql
    log_id UUID PRIMARY KEY,
    user_id INT REFERENCES user_profile(id),
    transaction_type VARCHAR(50) NOT NULL, -- 예: 'PURCHASE', 'DUNGEON_CLEAR', 'SHOP_SALE'
    amount BIGINT NOT NULL, -- 증감된 재화 양 (양수/음수로 처리 가능)
    currency_type VARCHAR(50),
    source_location VARCHAR(100), -- 어디서 발생했는지 추적 (예: /dungeon/level3)
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    ```

# ✨ Top 3 Game Concept & Design Briefs (v1.0)

## 💎 컨셉 1: 에테르 연대기 (Aether Chronicle)
*   **장르:** 하이 판타지 / 스토리 기반 퍼즐 RPG (Story-driven Puzzle RPG)
*   **로그라인:** 잊힌 고대의 기록(에테르 조각)을 모아 세계의 진실을 파헤치는 탐사자들의 여정. 단순 전투가 아닌 '논리적 해석'이 성장의 핵심이다.
*   **핵심 메커니즘 (Core Loop):**
    1.  **탐험 & 힌트 수집:** 플레이어는 지역 지도를 탐험하며 고대 유물이나 문양(에테르 조각)을 발견합니다.
    2.  **퍼즐 해결 (The Puzzle):** 수집한 에테르 조각들은 퍼즐의 재료가 됩니다. 이 조각들을 조합하여 '고대의 진실'이라는 스토리적 힌트를 얻어냅니다. (전투보다 해석이 중요).
    3.  **시스템 개방 (Key Unlocking):** 풀린 퍼즐은 다음 구역의 비밀 통로나, 캐릭터에게 새로운 스킬(Lore-based Skill)을 부여합니다. 이것이 성장의 주된 동기입니다.
    4.  **전투:** 전투는 비교적 단순한 턴제 시스템이지만, '에테르 조각' 조합으로 적의 약점을 논리적으로 공략하는 요소가 결합됩니다. (직접적인 스펙 싸움 지양).
*   **비즈니스 연결고리 (Monetization Focus):** **[정보/스토리 구매]**. 다음 퍼즐을 풀기 위한 '해석의 열쇠(Lore Key)'나, 시간 제한이 걸린 복잡한 고대 문명 패턴 분석 자료를 유료 판매합니다.
*   **🎨 비주얼 스타일 (Art Style Brief):**
    *   **톤앤매너:** 신비롭고 지적인 분위기. 장엄함과 아카이브(Archive) 느낌.
    *   **컬러 팔레트:**
        *   메인 컬러: #1A237E (딥 네이비/밤하늘) - 배경, 미스터리
        *   강조색 1: #FFC107 (골드 앰버) - 핵심 정보, 발견, 에테르 조각
        *   보조색: #BDBDBD (옅은 돌 회색) - 고대 유물, 지도의 질감
    *   **타이포그래피:** 세리프체 기반의 클래식하고 학술적인 폰트 조합. 제목/핵심 정보는 골드 강조와 함께 굵고 장엄하게 처리하여 '진실'의 느낌을 부여합니다.

## 💎 컨셉 2: 크로노스 게이트 (Chronos Gate)
*   **장르:** SF 생존 / 시간 역학 퍼즐 RPG (Time-Loop Puzzle Survival RPG)
*   **로그라인:** 시간이 반복되는 고대 폐허에 떨어진 탐사자. 시간을 조작하며 과거의 비밀을 파헤치고, 문명이 남긴 기술적 난제들을 해결해야 한다.
*   **핵심 메커니즘 (Core Loop):**
    1.  **시간 루프 진입:** 플레이는 일정한 시간 간격(예: 30분)으로 반복되는 스테이지에 갇힙니다. 각 루프에서 다른 행동을 시도할 수 있습니다.
    2.  **상호작용 및 기록:** 루프 A에서 발견한 자원이나 장치 배치는, 다음 루프 B의 환경 변화(예: 문이 열림/닫힘)를 유발합니다. 플레이어는 '시간 기록'이라는 형태로 경험치를 쌓습니다.
    3.  **기술 조합 퍼즐:** 단순히 스펙을 올리는 것이 아니라, 발견한 고대 기술 조각 A와 B를 결합하여 새로운 장치(예: 시간 정지 필드)를 만들어내는 방식이 핵심입니다.
    4.  **전투:** 적의 패턴과 약점을 '시간 흐름' 분석을 통해 파악하고, 가장 효율적인 순간에 제어된 스킬을 발동하는 전략성이 강조됩니다.
*   **비즈니스 연결고리 (Monetization Focus):** **[시스템 개선/정보 구매]**. 시간 기록 횟수 증가(루프 반복 횟수 늘리기), 고대 기술의 작동 원리 해독 가이드, 혹은 실패 시 재시도 기회 패키지 등을 판매합니다.
*   **🎨 비주얼 스타일 (Art Style Brief):**
    *   **톤앤매너:** 차갑고 미니멀하며 과학적인 느낌. 녹슨 금속과 빛의 대비가 중요함.
    *   **컬러 팔레트:**
        *   메인 컬러: #263238 (짙은 슬레이트 그레이) - 폐허, 기계 장치 배경
        *   강조색 1: #00BCD4 (시안 블루/네온 아쿠아) - 시간 흐름, 에너지, 핵심 기술 작동 시 빛.
        *   보조색: #8D6E63 (앤티크 브라운) - 고대 금속의 질감
    *   **타이포그래피:** 산세리프체 기반으로 깔끔하고 기능적인 느낌. 정보 패널이나 인터페이스는 HUD(Heads-Up Display)처럼 디자인하여 SF적 몰입도를 극대화합니다.

## 💎 컨셉 3: 그림자 장인 (The Shadow Weaver)
*   **장르:** 다크 판타지 / 관계 기반 퍼즐 액션 RPG (Relationship Puzzle Action RPG)
*   **로그라인:** 사람의 '감정'이나 '기억'을 재료로 삼아 괴물(그림자)에 맞서는 장인. 적을 물리치는 것보다, 그 근원을 이해하고 치유하는 것이 성장이다.
*   **핵심 메커니즘 (Core Loop):**
    1.  **감정 포착:** 플레이어는 스토리 속 NPC나 환경에서 특정 '감정의 조각' (슬픔, 분노, 기쁨 등)을 수집합니다. 이 감정이 퍼즐의 재료가 됩니다.
    2.  **그림자 연출 (Weaving):** 수집된 감정을 조합하여 시각적인 패턴(실타래/문양)을 만들고, 이를 이용하여 그림자로 이루어진 적에게 논리적 충격을 가합니다.
    3.  **관계 개선:** 퍼즐의 해답은 '힘'이 아니라 '공감과 이해'입니다. 특정 NPC와의 관계를 발전시키면 새로운 감정 재료나 스킬 트리가 개방됩니다. (성장의 열쇠가 바로 스토리라인에 있음).
    4.  **전투:** 액션성이 가미된 턴제 전투로, 적의 약점은 '물리적'이 아니라 '감정적 모순'입니다. 이 모순을 찾아내어 퍼즐처럼 공략합니다.
*   **비즈니스 연결고리 (Monetization Focus):** **[스토리 깊이/관계성 확보]**. 캐릭터들의 과거사(Lore)에 대한 추가 정보, 관계의 비밀을 푸는 '공감력 강화' 패키지 등 내러티브 소비를 유도합니다.
*   **🎨 비주얼 스타일 (Art Style Brief):**
    *   **톤앤매너:** 몽환적이고 어둡지만 따뜻한 대비가 강조되는 분위기. 동화적이면서도 성숙함.
    *   **컬러 팔레트:**
        *   메인 컬러: #424242 (어두운 그레이/먹물) - 그림자, 미스터리
        *   강조색 1: #E91E63 (딥 마젠타) - 감정, 핵심 에너지(감정을 시각화한 색).
        *   보조색: #FFD700 (따뜻한 골드) - 기억, 빛, 희망의 상징.
    *   **타이포그래피:** 손글씨 느낌이나 필기체가 가미된 폰트와 조합하여 '개인적인 기록', '이야기' 같은 아날로그적 감성을 부여합니다.

# 📜 [Aether Chronicle] 핵심 스토리 및 세계관 구축 (Story Pitch)

## 💎 로그라인 (Logline):
> 고대의 진실은 힘이 아니라, 논리에서 온다. 세상의 가장 깊은 비밀을 해독하는 마지막 탐험가의 기록.

## 📖 배경 이야기 초안 (World Lore Draft):
(스토어 소개나 게임 내 인트로 영상에 활용할 수 있는 서사입니다.)

**"시간은 흐르고, 역사는 반복되지만, 진실은 항상 미궁 속에 숨겨져 있었다."**

태초부터 인간들은 '에테르'라 불리는 근원적 에너지의 존재를 믿었다. 에테르는 모든 생명과 문명을 움직이는 힘이자, 동시에 가장 강력한 비밀 그 자체였다. 고대 문명의 최정점에서도 이 에테르의 원리를 완전히 이해하지 못했고, 수많은 미스터리와 대재앙을 낳았다.

우리가 사는 세계는 거대한 '기록체계(Archive)'에 의해 지탱되고 있지만, 그 기록은 조각나고, 중요한 페이지들은 봉인되어 있다. 사람들은 이를 단순한 역사의 흐름이라 치부하지만, 진짜 진실은 **'에테르 조각(Aether Fragments)'**이라는 형태로 남아있다. 이 조각들을 모으고, 고대 언어의 퍼즐을 풀어나가야만 비로소 봉인된 지식과 세계를 재건할 수 있다.

당신은 바로 그 미지의 진실을 찾아 나서는 **최후의 탐험가(The Last Archaeologist)**다.
단순한 전투력이나 마법으로는 도달할 수 없는, 역사의 심장부로 발을 들여놓게 될 것이다. 당신이 풀어내는 하나의 퍼즐 조각은, 잊힌 문명 전체를 움직이는 거대한 서사적 동력이 된다.

# 🚀 hahasoft 게임 MVP 개발 로드맵 (Minimum Viable Product) - [수정본: Business Priority]

## 🎯 목표 및 범위
*   **최종 목표:** 모바일 RPG를 통해 $1M 수익 달성
*   **MVP 핵심 정의:** 시장성이 검증된 코어 루프(Core Loop)와 **가장 빠르게 재무적 가치를 증명할 수 있는 핵심 과금 재미 요소**에 집중한다.

---

## 🗺️ 개발 로드맵 (Phase I: MVP Focus - Revenue Validation Priority)

### 1. [최우선] Core Gameplay & Monetization Hook (핵심 수익 검증 루프)
*   **목표:** 유저가 자연스럽게 과금의 필요성을 느끼는 최소한의 순환 구조를 구현한다.
*   **범위:**
    *   메인 화면 및 캐릭터 기본 성장창: **'성장 정체 구간(Pain Point)'이 반드시 발생하도록 설계.** (→ Monetization Trigger)
    *   퍼즐 해결 기반 성장 시스템: 가장 기본적인 레벨업/재화 획득만 구현.
    *   **핵심 수익 요소 도입:** 유료 재화 구매를 통해 **'시간 단축' 또는 '성장 정체 해소(Skip)'가 가능한 기능**을 반드시 연결. (예: 프리미엄 부스터, 즉시 경험치 패키지)
*   **KPI 검증 항목:**
    1.  **D7 Retention Rate:** Core Loop를 돌리는 데 성공한 유저 비율.
    2.  **Conversion Point Ratio:** 어느 단계(어떤 성장의 정체 구간)에서 과금 결정을 내리는가? (핵심 데이터)
    3.  **ARPDAU (Average Revenue Per Daily Active User):** 초기 광고/과금 유도 효율성 측정.

### 2. [필수] Growth System & Retention Loop (유저 유지 시스템)
*   **목표:** 유저가 꾸준히 돌아오게 만드는 동기 부여 구조를 완성한다. 과금이 없는 상태에서도 즐길 거리를 제공하여 이탈을 막는다.
*   **범위:**
    *   랭킹 시스템(Leaderboard): 경쟁심을 자극하는 최소한의 랭킹 기능 구현. (핵심: 상위에 오르기 위해 시간이 필요한 구조)
    *   일일/주간 보상 시스템: 반복 플레이를 유도하는 최소한의 루틴 제공.
    *   **전략적 고려:** 이 로직은 과금을 통해 *빠르게 최고 등급에 도달하게 만드는 배경*으로만 활용하고, 자체적으로 재미가 너무 강해지면 수익화 기회를 잃을 수 있으므로 기능 범위를 엄격히 제한한다.

### 3. [고도화] 추가 콘텐츠 및 시스템 (Phase II 이관)
*   **목표:** MVP 검증 후, 매출 극대화를 위한 확장 영역으로 정의하고 개발 우선순위에서 배제한다.
*   **범위:**
    *   스토리 분기점/캐릭터 커스터마이징: **MVP에서는 '외형'만 기본 제공.** 스토리라인은 최소한의 목표 달성에 필요한 메커니즘 설명에 그친다. (→ 개발 리소스 절감)
    *   소셜 기능(친구 초대, 길드): MVP 단계에서 구현할 필요가 없다. 초기에는 개인 성장과 과금 자체에 집중한다.

**✅ 현빈의 최종 결론:**
MVP는 **'성장 정체 → 해결책 구매 → 성취감 획득 → 다음 정체 구간으로 이동'** 이라는 수익화 루프를 검증하는 데 모든 리소스를 투입해야 합니다. 다른 기능들은 일단 '더 나중에(Later)'로 미룹니다.
